# 🚀 CELEBRA V4 - Deployment Guide\n\n## 📋 Pré-requisitos\n\n### Infraestrutura\n- [ ] Servidor Linux (Ubuntu 22.04+)\n- [ ] Docker & Docker Compose\n- [ ] Node.js 22+\n- [ ] MySQL 8.0+\n- [ ] Redis 7+\n- [ ] Nginx (reverse proxy)\n\n### Contas & Serviços\n- [ ] AWS Account (S3, CloudWatch)\n- [ ] Firebase Account (Push Notifications)\n- [ ] Sentry Account (Error Tracking)\n- [ ] GitHub Account (CI/CD)\n- [ ] Domain Name\n- [ ] SSL Certificate (Let's Encrypt)\n\n### Credenciais\n- [ ] OAuth Providers (Google, Facebook, Manus)\n- [ ] Stripe Keys\n- [ ] Spotify API Keys\n- [ ] YouTube API Keys\n- [ ] TikTok API Keys\n- [ ] Email SMTP Credentials\n\n---\n\n## 🔧 Configuração de Ambiente\n\n### 1. Preparar Servidor\n\n```bash\n# Update system\nsudo apt update && sudo apt upgrade -y\n\n# Install dependencies\nsudo apt install -y curl wget git build-essential\n\n# Install Node.js\ncurl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -\nsudo apt install -y nodejs\n\n# Install pnpm\ncurl -fsSL https://get.pnpm.io/install.sh | sh -\n\n# Install Docker\ncurl -fsSL https://get.docker.com -o get-docker.sh\nsudo sh get-docker.sh\n\n# Install Docker Compose\nsudo curl -L \"https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)\" -o /usr/local/bin/docker-compose\nsudo chmod +x /usr/local/bin/docker-compose\n```\n\n### 2. Clone Repository\n\n```bash\ncd /opt\nsudo git clone https://github.com/marcodivulga-design/celebra-v4.git\ncd celebra-v4\nsudo chown -R $USER:$USER .\n```\n\n### 3. Setup Environment Variables\n\n```bash\n# Copy example file\ncp .env.example .env\n\n# Edit with production values\nnano .env\n\n# Set permissions\nchmod 600 .env\n```\n\n### 4. Install Dependencies\n\n```bash\npnpm install --frozen-lockfile\n```\n\n---\n\n## 🐳 Docker Deployment\n\n### 1. Build Docker Image\n\n```bash\ndocker build -t celebra-v4:latest .\n```\n\n### 2. Create Docker Compose File\n\n```yaml\n# docker-compose.yml\nversion: \"3.8\"\n\nservices:\n  app:\n    image: celebra-v4:latest\n    ports:\n      - \"3000:3000\"\n    environment:\n      - NODE_ENV=production\n      - DATABASE_URL=${DATABASE_URL}\n      - REDIS_URL=${REDIS_URL}\n      - JWT_SECRET=${JWT_SECRET}\n    depends_on:\n      - mysql\n      - redis\n    restart: always\n    healthcheck:\n      test: [\"CMD\", \"curl\", \"-f\", \"http://localhost:3000/health\"]\n      interval: 30s\n      timeout: 10s\n      retries: 3\n\n  mysql:\n    image: mysql:8.0\n    environment:\n      - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}\n      - MYSQL_DATABASE=${MYSQL_DATABASE}\n    volumes:\n      - mysql_data:/var/lib/mysql\n    restart: always\n\n  redis:\n    image: redis:7-alpine\n    restart: always\n    volumes:\n      - redis_data:/data\n\n  nginx:\n    image: nginx:latest\n    ports:\n      - \"80:80\"\n      - \"443:443\"\n    volumes:\n      - ./nginx.conf:/etc/nginx/nginx.conf:ro\n      - ./ssl:/etc/nginx/ssl:ro\n    depends_on:\n      - app\n    restart: always\n\nvolumes:\n  mysql_data:\n  redis_data:\n```\n\n### 3. Start Services\n\n```bash\ndocker-compose up -d\n\n# Check status\ndocker-compose ps\n\n# View logs\ndocker-compose logs -f app\n```\n\n---\n\n## 🔐 SSL/TLS Setup\n\n### 1. Generate SSL Certificate (Let's Encrypt)\n\n```bash\n# Install Certbot\nsudo apt install -y certbot python3-certbot-nginx\n\n# Generate certificate\nsudo certbot certonly --standalone -d celebra.app -d api.celebra.app\n\n# Auto-renewal\nsudo systemctl enable certbot.timer\n```\n\n### 2. Configure Nginx\n\n```nginx\n# nginx.conf\nupstream app {\n  server app:3000;\n}\n\nserver {\n  listen 80;\n  server_name celebra.app api.celebra.app;\n  return 301 https://$server_name$request_uri;\n}\n\nserver {\n  listen 443 ssl http2;\n  server_name celebra.app api.celebra.app;\n\n  ssl_certificate /etc/nginx/ssl/fullchain.pem;\n  ssl_certificate_key /etc/nginx/ssl/privkey.pem;\n  ssl_protocols TLSv1.2 TLSv1.3;\n  ssl_ciphers HIGH:!aNULL:!MD5;\n\n  # Security headers\n  add_header Strict-Transport-Security \"max-age=31536000; includeSubDomains\" always;\n  add_header X-Frame-Options \"DENY\" always;\n  add_header X-Content-Type-Options \"nosniff\" always;\n  add_header X-XSS-Protection \"1; mode=block\" always;\n\n  # Proxy to app\n  location / {\n    proxy_pass http://app;\n    proxy_set_header Host $host;\n    proxy_set_header X-Real-IP $remote_addr;\n    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;\n    proxy_set_header X-Forwarded-Proto $scheme;\n    proxy_read_timeout 90;\n  }\n\n  # WebSocket support\n  location /ws {\n    proxy_pass http://app;\n    proxy_http_version 1.1;\n    proxy_set_header Upgrade $http_upgrade;\n    proxy_set_header Connection \"upgrade\";\n    proxy_set_header Host $host;\n    proxy_set_header X-Real-IP $remote_addr;\n    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;\n  }\n}\n```\n\n---\n\n## 🗄️ Database Setup\n\n### 1. Create Database\n\n```bash\nmysql -u root -p\n\nCREATE DATABASE celebra_prod CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;\nCREATE USER 'celebra'@'localhost' IDENTIFIED BY 'strong_password';\nGRANT ALL PRIVILEGES ON celebra_prod.* TO 'celebra'@'localhost';\nFLUSH PRIVILEGES;\nEXIT;\n```\n\n### 2. Run Migrations\n\n```bash\npnpm drizzle-kit migrate\n```\n\n### 3. Seed Initial Data\n\n```bash\npnpm seed\n```\n\n---\n\n## 📊 Monitoring Setup\n\n### 1. Configure Sentry\n\n```bash\n# Already configured in .env\nSENTRY_DSN=https://key@sentry.io/project\n```\n\n### 2. Setup CloudWatch Logs\n\n```bash\n# Install CloudWatch agent\nwget https://s3.amazonaws.com/amazoncloudwatch-agent/ubuntu/amd64/latest/amazon-cloudwatch-agent.deb\nsudo dpkg -i -E ./amazon-cloudwatch-agent.deb\n\n# Configure agent\nsudo /opt/aws/amazon-cloudwatch-agent/bin/amazon-cloudwatch-agent-config-wizard\n\n# Start agent\nsudo /opt/aws/amazon-cloudwatch-agent/bin/amazon-cloudwatch-agent-ctl \\\n  -a fetch-config \\\n  -m ec2 \\\n  -s\n```\n\n### 3. Setup Prometheus & Grafana\n\n```bash\n# Add to docker-compose.yml\nprometheus:\n  image: prom/prometheus:latest\n  volumes:\n    - ./prometheus.yml:/etc/prometheus/prometheus.yml\n    - prometheus_data:/prometheus\n\ngrafana:\n  image: grafana/grafana:latest\n  ports:\n    - \"3001:3000\"\n  environment:\n    - GF_SECURITY_ADMIN_PASSWORD=admin\n  volumes:\n    - grafana_data:/var/lib/grafana\n```\n\n---\n\n## 🧪 Pre-Deployment Testing\n\n### 1.
